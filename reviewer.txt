Below is a comprehensive list of commands used throughout the development of this project, organized by category. These include environment setup, dependency management, database migrations, running the server, testing, seeding data, and Docker.

---

### 1. **Project Setup & Environment**

| Command | Purpose |
|---------|---------|
| `git clone <repo-url>` | Clone the repository |
| `cd ClinicManagementCore` | Enter project directory |
| `python -m venv venv` | Create virtual environment |
| `source venv/bin/activate` (Linux/Mac) or `venv\Scripts\activate` (Windows) | Activate virtual environment |
| `pip install -r requirements.txt` | Install dependencies |

---

### 2. **Environment Variables (.env)**

| Command / File | Purpose |
|----------------|---------|
| Create `.env` file | Store sensitive configuration |
| `DATABASE_URL=postgresql+asyncpg://...` | Async PostgreSQL URL |
| `DATABASE_URL=sqlite+aiosqlite:///./clinic.db` | Async SQLite URL (development) |
| `SECRET_KEY=...` | JWT secret key |
| `ALGORITHM=HS256` | JWT algorithm |
| `ACCESS_TOKEN_EXPIRE_MINUTES=30` | Token expiry |
| `ENV=development` | Environment flag |

---

### 3. **Database Migrations (Alembic)**

| Command | Purpose |
|---------|---------|
| `alembic init alembic` | Initialize Alembic (creates `alembic/` folder and `alembic.ini`) |
| `alembic revision --autogenerate -m "initial"` | Generate migration script from models |
| `alembic upgrade head` | Apply all migrations |
| `alembic downgrade base` | Revert all migrations (if needed) |

---

### 7. **Docker & Docker Compose**

| Command | Purpose |
|---------|---------|
| `docker-compose up --build` | Build and start all services (API, PostgreSQL, pgAdmin) |
| `docker-compose up -d` | Run in detached mode |
| `docker-compose down` | Stop and remove containers |
| `docker-compose exec api alembic upgrade head` | Run migrations inside the API container |
| `docker-compose exec api python scripts/seed_dev_data.py` | Seed data inside the container |
| `docker-compose logs -f api` | Follow API logs |
